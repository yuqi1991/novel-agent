# 前端架构

本文描述 Web UI 的页面关系、组件边界和测试策略。

## 页面关系

当前只有本地单用户 Web UI，没有账号、团队或多人联机页面。

| 路由 | 作用 |
| --- | --- |
| `/` | 首页和故事库抽屉入口。默认展示故事库；如果没有故事，进入新建故事模式。 |
| `/stories/[storyId]` | 故事游玩工作区。默认加载该故事最后一次存档；没有存档则创建默认存档。 |
| `/stories/[storyId]?panel=<panel>` | 在聊天工作区右侧打开对应管理面板。 |

## 顶部导航和抽屉

故事页顶部横向标题栏包含这些入口：

- 故事库
- 故事资料 / 世界书
- Agent 管理
- Agent 编排
- 存档管理
- 运行记录

点击入口后，聊天窗口保持在中间，管理内容在右侧抽屉显示。这个交互参考 SillyTavern：核心游玩区不被管理表单替代，故事库和资料编辑通过侧边面板进入。

## 故事库抽屉

`StoryLibraryDrawerClient` 负责三种模式：

1. **故事列表**：先展示所有故事。顶部是一排小按钮，用于切换“列表 / 新建 / 导入 SillyTavern”。
2. **新建故事**：进入故事资料草稿编辑器。只有点击保存后才创建故事和默认存档。
3. **导入 SillyTavern**：解析角色卡或世界书后进入同一份故事资料草稿，并用解析出的角色、世界书和第一条消息预填。

点击已有故事时：

- 中间聊天窗口切换到该故事的最后一次存档。
- 右侧打开“故事资料 / 世界书”面板，方便继续编辑资料。

## 故事资料面板

故事资料面板复用现有 Story Material 服务，不创建第二套资料存储。

它对应的数据包括：

- Story title / description。
- Character Profiles。
- World Entries。
- optional Player Character。
- 未来的 opening message 字段。

当前 MVP 没有第一条消息的数据库字段；UI 可以显示占位区，但持久化需要单独 schema 和服务变更。

## 聊天工作区

聊天工作区包含：

- 当前故事和当前存档信息。
- 原始聊天记录。
- 最新 Agent 回复的重掷和变体选择。
- 玩家输入框。

只有最新的 Agent 回复楼可以原地重掷或切换变体。历史楼层只能用于 fork 新存档。

## 存档管理面板

存档管理绑定到当前故事，展示：

- 该故事下的所有 Play Sessions。
- 当前存档聊天记录。
- Progress Wiki 文件浏览/编辑区域。
- Wiki Snapshot 操作。

Progress Wiki 不放在聊天页右上角，因为它属于具体存档。

## UI 测试边界

Playwright 主要覆盖：

- 故事库渲染、新建故事、打开已有故事。
- SillyTavern 导入并预填故事资料。
- 聊天、重掷、变体选择、分叉。
- 故事资料、世界书和存档管理面板是否可达。

服务层已经验证过的复杂数据规则，不在 e2e 中重复做低层断言。
